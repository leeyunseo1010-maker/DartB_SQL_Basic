## 2-3. 데이터 탐색 (SELECT, FROM, WHERE)

~~~
✅ 학습 목표 :
* SQL 쿼리 구조를 이해할 수 있다. 
* SELECT, FROM, WHERE의 핵심 문법을 설명할 수 있다. 
~~~

### SQL 쿼리의 기본 구조

SQL 쿼리는 **SELECT**, **FROM**, **WHERE** 세 가지 키워드로 구성되어 있습니다. 쿼리문을 작성할 때의 순서는 `SELECT → FROM → WHERE` 이지만, 실제로 쿼리 엔진이 내부적으로 실행하는 순서는 `FROM → WHERE → SELECT` 입니다. 따라서 쿼리를 작성할 때 인지적으로도 "어디에서(FROM) → 어떤 조건으로(WHERE) → 무엇을 출력할지(SELECT)" 순서로 생각하면 이해하기 쉽다고 판단하였습니다.

### FROM

FROM은 **어떤 테이블에서 데이터를 확인할 것인가**를 명시하는 키워드입니다. BigQuery에서는 `dataset.table` 형태로 입력합니다. 프로젝트가 여러 개인 경우에는 `프로젝트ID.데이터셋.테이블` 형태로 명시하는 것이 좋습니다. 프로젝트가 단일이라면 프로젝트 ID를 생략해도 실행이 가능합니다. 또한 테이블 이름이 길 경우 `AS 별칭`을 사용하여 간단하게 표현할 수 있습니다.

### WHERE

WHERE은 **FROM에 명시된 테이블에서 원하는 조건으로 데이터를 필터링**할 때 사용하는 키워드입니다. 예를 들어 `WHERE type1 = 'Fire'`라고 작성하면 type1 컬럼의 값이 Fire인 데이터만 가져오게 됩니다. 조건이 필요 없는 경우에는 WHERE절을 생략해도 실행이 가능합니다.

### SELECT

SELECT는 **테이블에서 어떤 컬럼을 선택하여 출력할 것인가**를 지정하는 키워드입니다. 컬럼을 여러 개 출력하고 싶을 때는 쉼표(,)로 구분하여 나열하면 됩니다. `AS`를 사용하면 출력되는 컬럼의 이름을 별칭으로 변경할 수 있습니다. 이때 별칭에는 따옴표를 넣지 않아야 한다는 점이 중요합니다.

### SELECT * 과 SELECT EXCEPT

`SELECT *`은 모든 컬럼을 출력하겠다는 의미입니다. 다만 BigQuery에서는 데이터 스캔량에 따라 비용이 발생하기 때문에, 모든 컬럼을 스캔하는 `SELECT *`은 비용 측면에서 비추천되는 안티 패턴이라고 하였습니다. 데이터를 확인하고 싶을 때는 BigQuery의 **미리보기** 기능을 활용하는 것이 좋습니다. 또한 특정 컬럼만 제외하고 보고 싶을 경우에는 `SELECT * EXCEPT(컬럼명)` 문법을 사용할 수 있습니다.

### 기타 학습 내용

- **세미콜론(;)** : 하나의 쿼리가 끝났음을 의미합니다. 여러 쿼리를 한 번에 실행할 때 세미콜론으로 구분합니다.
- **주석(#)** : 샵(#) 기호를 사용하거나 단축키(Cmd/Ctrl + /)로 주석 처리가 가능합니다. 주석은 쿼리 실행에 영향을 주지 않습니다.
- **가독성 있는 쿼리 작성** : 쿼리를 한 줄로 작성해도 실행은 되지만, 협업과 유지보수를 위해 키워드별로 줄을 나누어 가독성 있게 작성하는 것이 중요합니다.
- **쿼리 작성 순서 주의** : `SELECT → FROM → WHERE` 순서는 반드시 지켜야 합니다. `SELECT → WHERE → FROM` 등 순서가 바뀌면 실행되지 않습니다.


## 2-5. 집계 (Group By / HAVING / SUM,COUNT)

~~~
✅ 학습 목표 :
* 데이터를 집계하고 그룹화하는 방법을 설명할 수 있다.
* GROUP BY, HAVING, ORDER BY, 집계함수(SUM/COUNT 등)을 활용하는 방법을 설명할 수 있다.
* having과 where의 차이에 대해서 설명할 수 있다.
~~~

### 집계(Aggregation)란?

집계란 **모아서 계산하는 것**을 의미합니다. 여기서 "모으다"는 그룹화(GROUP BY)를, "계산하다"는 집계함수(COUNT, SUM, AVG 등)를 의미합니다. 원본 데이터에는 로우가 수만~수십만 건이 있을 수 있는데, 이를 특정 기준으로 그룹화하여 통계자료나 건수 등을 산출하는 과정이 집계라고 이해하였습니다.

### GROUP BY

GROUP BY는 **같은 값끼리 모아서 그룹화**하는 문법입니다. 예를 들어 포켓몬 데이터에서 `GROUP BY type1`을 하면 타입별로 데이터가 묶이게 됩니다. GROUP BY를 사용할 때 주의할 점은, GROUP BY에 명시한 컬럼을 SELECT에도 반드시 함께 명시해주어야 한다는 것입니다. 이 둘은 세트로 사용된다고 이해하였습니다.

### 집계함수

집계함수는 그룹화된 데이터에 대해 계산을 수행하는 함수입니다. 자주 사용되는 집계함수는 다음과 같습니다:

- **COUNT** : 로우의 개수를 세는 함수입니다.
- **SUM** : 값의 합계를 구하는 함수입니다.
- **AVG** : 값의 평균을 구하는 함수입니다.
- **MAX** : 최댓값을 구하는 함수입니다.
- **MIN** : 최솟값을 구하는 함수입니다.

### DISTINCT

DISTINCT는 **중복을 제거하고 고유한 값만 보고 싶을 때** 사용합니다. 예를 들어 `COUNT(DISTINCT user_id)`는 중복을 제외한 유니크한 유저 수를 세는 것이고, `COUNT(user_id)`는 중복을 포함한 전체 건수를 세는 것입니다. 중복 제거 시 DISTINCT와 GROUP BY 모두 활용이 가능하며, 상황에 따라 적절한 것을 선택하면 된다고 이해하였습니다.

### WHERE와 HAVING의 차이

WHERE와 HAVING은 모두 조건을 설정하는 키워드이지만 사용 시점이 다릅니다:

- **WHERE** : 테이블의 원본 데이터에 바로 조건을 설정할 때 사용합니다. 즉, **집계 전**에 필터링하는 것입니다. FROM절 바로 아래에 위치합니다.
- **HAVING** : GROUP BY로 집계한 **후**에 조건을 설정할 때 사용합니다. 집계함수로 생성된 컬럼에 조건을 걸고 싶을 때 HAVING을 사용합니다. GROUP BY절 아래에 위치합니다.

예를 들어 "포켓몬 수가 10마리 이상인 타입만 보고 싶다"라고 하면, 포켓몬 수는 COUNT로 집계한 결과이므로 HAVING을 사용해야 합니다. WHERE에서는 집계함수를 사용할 수 없다는 점이 핵심이라고 판단하였습니다.

### ORDER BY

ORDER BY는 **출력 결과를 정렬**할 때 사용합니다. 기본값(디폴트)은 오름차순(ASC)이며, 내림차순으로 정렬하고 싶을 때는 `DESC`를 명시합니다. ORDER BY는 쿼리문의 맨 마지막에 작성하며, 서브쿼리 내부에서는 사용할 필요가 없다고 이해하였습니다.

### LIMIT

LIMIT은 **출력되는 로우 수를 제한**하고 싶을 때 사용합니다. 쿼리문의 가장 마지막에 위치하며, ORDER BY 아래에 작성합니다.

### SQL 쿼리 전체 실행 순서 정리

최종적으로 쿼리문의 작성 형태와 실행 순서를 정리하면 다음과 같습니다:

~~~
작성 순서: SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY → LIMIT
실행 순서: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT
~~~

# 3️⃣ 확인문제

## 문제 1

> **🧚Q. 포켓몬 마스터 진아는 포켓몬 데이터 조회하는 SQL문에 재미를 느껴서 혼자서 데이터를 조회하는 쿼리문을 짰습니다. 하지만 세 가지의 오류로 다음 코드가 실행이 안된다고 하는데, 각 오류의 위치와 이유를 설명하고, 올바른 쿼리문으로 수정해보세요.**

~~~sql
# 진아의 SQL Query문 
SELECT name. type
FROM pokemon;
WHERE type = Electric;
~~~



~~~
오류 1 : SELECT name. type에서 name과 type 사이에 마침표(.)가 사용되었습니다.
→ 여러 컬럼을 나열할 때는 마침표가 아닌 쉼표(,)를 사용해야 합니다.

오류 2 : FROM pokemon; 에서 세미콜론(;)이 붙어 있습니다.
→ 세미콜론은 하나의 쿼리가 끝났음을 의미하기 때문에, FROM pokemon 뒤에 세미콜론이 있으면
  WHERE절이 별도의 쿼리로 인식되어 실행되지 않습니다. 세미콜론을 제거해야 합니다.

오류 3 : WHERE type = Electric; 에서 Electric에 따옴표가 없습니다.
→ 문자열 값을 조건으로 사용할 때는 작은따옴표('')로 감싸주어야 합니다.
  숫자가 아닌 문자열(String)을 필터링할 때는 반드시 따옴표가 필요합니다.

올바르게 수정한 쿼리문은 다음과 같습니다:
~~~

~~~sql
SELECT name, type
FROM pokemon
WHERE type = 'Electric';
~~~



## 문제 2

> **🧚Q. 앞서 SQL Query의 오류를 해결한 진아는 기분 좋게 이번에는 포켓몬 데이터에서 타입별 평균 공격력이 60 이상인 타입만 조회하려는 쿼리를 작성하려고 했습니다. 하지만 이번에도 실수를 하여 쿼리문이 실행되지 않거나 잘못된 결과가 나오고 있는데, 쿼리에서 잘못된 부분이 무엇인지 설명하고, 올바르게 수정한 쿼리를 작성해보세요.**

~~~sql
SELECT type, AVG(attack) AS avg_attack
FROM pokemon
WHERE AVG(attack) >= 60
GROUP BY type;
~~~



~~~
오류 : WHERE절에 집계함수 AVG(attack)를 사용한 것이 잘못되었습니다.
→ WHERE는 테이블의 원본 데이터에 바로 조건을 설정하는 키워드이기 때문에,
  집계함수(AVG, COUNT, SUM 등)를 WHERE절에서 사용할 수 없습니다.
  집계함수로 계산된 결과에 조건을 설정하고 싶을 때는 GROUP BY 이후에
  HAVING을 사용해야 합니다.

강의에서 배운 내용에 따르면, WHERE는 "집계 전"에 원본 테이블에 조건을 거는 것이고,
HAVING은 "집계 후"에 그룹화된 결과에 조건을 거는 것입니다.
따라서 AVG(attack) >= 60 이라는 조건은 GROUP BY로 집계한 후의 결과에
적용해야 하므로 HAVING을 사용하는 것이 올바릅니다.

올바르게 수정한 쿼리문은 다음과 같습니다:
~~~

~~~sql
SELECT type, AVG(attack) AS avg_attack
FROM pokemon
GROUP BY type
HAVING AVG(attack) >= 60;
~~~
